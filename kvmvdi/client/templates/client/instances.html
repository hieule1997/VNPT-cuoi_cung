New OpenStack Instance

You can create a new Instance by first choosing a name then selecting the Region where the instance will be located, the source to boot from, a configuration (flavor) and the authentication parameters.

Boot Source Selection

Select the image to boot from, from the list of available images. You can also boot from an instance snapshot if any is available. The option Create new volume while selecting an image will create a new volume from the selected image and will start the Instance with the volume as the main root disk You can also boot directly from an existing volume.

Auto delete volume when instance is terminated will mark the volume to be automatically deleted when the Instance is terminated. If this option is not set, the volume will remain active even after you remove the instance and can be managed from the Volumes menu.

Select a configuration In order to create the new Instance you also need to select a configuration.
